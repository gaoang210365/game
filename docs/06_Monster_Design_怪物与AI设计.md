# 《回声病房》怪物与 AI 设计 / Monster & AI Design

> **文档版本：** 0.1（初稿）
> **前置文档：** 01～05 号文档
> **文档日期：** 2026-07-14
> **文档状态：** 怪物设计，供 AI 实现参考

---

## 1. 目的

本文档定义主要怪物“值夜护士”的外观、行为、感知、AI 状态机与参数框架，以及非实体威胁的行为规则，确保怪物设计服务于心理恐怖体验，并保持公平性。

设计总原则：

- 怪物是压力来源，不是纯粹的击杀目标；
- 感知基于声音、光线、视线，规则透明；
- 不作弊、不穿墙、不瞬移；
- 玩家可通过学习提高生存率；
- 怪物设计与剧情主题（责任与自我审判）统一。

---

## 2. 主要怪物：值夜护士

### 2.1 概念定位

值夜护士既是事故中death的护士周宁的投影，也是主角林澈无法面对的责任本身。随着循环推进，其形象逐渐与主角重合，暗示“怪物即自我审判”。

### 2.2 外观

- 身形近似人类，四肢比例存在轻微错误；
- 面部被过曝白光、纱布或阴影遮挡；
- 穿着不属于同一年代的护士制服，无医院标志；
- 拖着输液架，输液瓶发出可辨认碰撞声；
- 静止时接近场景道具，移动时动作不连续；
- 终章细节逐渐显现与主角的相似性。

### 2.3 标识音

- 输液瓶玻璃碰撞（核心识别音）；
- 异常鞋跟声；
- 失真呼吸；
- 接近时环境底噪减弱。

标识音是玩家判断怪物位置和距离的主要依据，必须清晰且方向可辨。

---

## 3. 感知系统

### 3.1 感知来源

- 听觉：玩家噪声（移动、开门、碰撞）与环境声源；
- 视觉：视线锥内的玩家；
- 光线：手电筒光锥提高被发现概率。

### 3.2 听觉模型

- 玩家噪声值随距离衰减；
- 环境声源可部分掩盖玩家噪声；
- 高噪声动作（奔跑、碰撞、开门）产生瞬时高感知；
- 怪物向可疑声源移动，而非直接锁定玩家。

### 3.3 视觉模型

- 有限视线锥与视距；
- 阴影、遮挡物降低被发现概率；
- 手电筒光锥在视线内显著提高发现概率；
- 玩家蹲伏、静止降低视觉暴露。

### 3.4 光线模型

- 开灯移动最容易暴露；
- 关灯降低暴露但影响玩家自身视野；
- 部分段落怪物会被光主动吸引。

## 4. AI 状态机

### 4.1 状态

```text
巡逻(Patrol)
察觉(Suspicious)
调查(Investigate)
追逐(Chase)
搜索(Search)
失去目标(LostTarget)
返回(Return)
```

### 4.2 状态转换

```text
Patrol --听到/看到异常--> Suspicious
Suspicious --确认可疑源--> Investigate
Investigate --发现玩家--> Chase
Investigate --无发现--> Return
Chase --失去视线/声音--> Search
Search --找到玩家--> Chase
Search --超时未找到--> LostTarget
LostTarget --> Return
Return --回到巡逻路径--> Patrol
```

### 4.3 各状态行为

- Patrol：沿预设路径巡逻，感知灵敏度中等；
- Suspicious：停顿、转向可疑方向，灵敏度提高；
- Investigate：前往可疑位置，搜索周边；
- Chase：追击玩家，速度提升，持续更新玩家位置（仅在有视线/声音时）；
- Search：前往玩家最后已知位置，进行范围搜索；
- LostTarget：搜索超时，逐渐降低警戒；
- Return：返回巡逻路径或切换新路径。

---

## 5. 公平性规则

- 不无提示瞬移到玩家身后；
- 不穿墙、不穿门（撞门有动画与声音）；
- 不永久知道玩家位置，仅在有感知依据时更新；
- 追逐前必有预警（标识音或视觉）；
- 追逐路线经灰盒测试，提供至少两个决策点；
- 躲藏不是绝对安全，但正确行为应能脱身；
- 难度调节通过感知范围、移动速度、搜索时间实现，不破坏规则一致性。

---

## 6. 参数框架

以下为可调参数（数值待灰盒测试确定）：

| 参数 | 说明 |
|---|---|
| 巡逻速度 | 基础移动速度 |
| 追逐速度 | 追击时速度，略高于玩家奔跑或相近 |
| 视距 | 视觉发现最大距离 |
| 视野角 | 视线锥角度 |
| 听觉半径 | 基础可听噪声范围 |
| 光线加成 | 手电筒对被发现概率的提升 |
| 调查时间 | Investigate 停留时间 |
| 搜索时间 | Search 持续时间 |
| 记忆时长 | 失去目标后仍记住最后位置的时间 |
| 循环记忆强度 | 随循环推进对玩家常用路线的熟悉度 |

参数按难度预设分组（低/标准/高）。

## 7. 与循环系统结合

- 怪物在循环推进中逐渐“记住”玩家常用路线，用于叙事与渐进压力；
- 循环记忆表现为：更频繁巡逻玩家常走区域、更快前往常用躲藏点；
- 记忆强度有上限，避免后期完全无法通过；
- 终章循环崩坏时，怪物行为可脱离常规巡逻，进入脚本化最终追逐。

---

## 8. 追逐段落中的怪物行为

### D01 首次追逐（教学）
- 感知灵敏度适中，给玩家学习空间；
- 搜索时间较短；
- 明确的逃生路线与躲藏点。

### D02 第二次追逐（环境利用）
- 灵敏度提高；
- 需要玩家使用环境机关（电源、声源）；
- 怪物会被声源吸引，允许玩家调虎离山。

### D03 最终追逐（终章）
- 脚本化演出为主；
- 结合空间崩坏与最终谜题；
- 允许打破部分常规巡逻规则，但仍提供玩家可行的通过路径。

---

## 9. 非实体威胁行为

### 9.1 移动的病床/轮椅
- 仅在玩家视线外移动；
- 不直接造成死亡，制造压力与阻碍。

### 9.2 镜像
- 模仿玩家动作，存在时间差；
- 用于心理压力与叙事暗示。

### 9.3 假人
- 未被照亮时靠近；
- 被照亮时静止；
- 首版定位为心理压力，是否致命待确认。

### 9.4 错误广播
- 引发空间变化；
- 不直接攻击玩家。

---

## 10. 死亡与恢复

- 被怪物抓住触发失败流程，从最近检查点恢复；
- 失败可穿插短暂记忆片段，提供额外叙事；
- 连续失败可适度降低搜索时间，但不明显提示动态难度；
- 恢复不强制重复长篇阅读与已完成简单操作。

---

## 11. 实现建议

- 先实现可视化状态机与调试显示（当前状态、感知值、目标位置）；
- 在灰盒地图中单独测试巡逻、感知、追逐、搜索；
- 使用导航网格或路径点系统控制移动；
- 感知系统与玩家噪声/光照系统通过事件解耦；
- 参数集中在配置文件，便于调试。

---

## 12. 待确认事项

1. 追逐速度是否略高于玩家，还是依靠走位取胜；
2. 假人是否致命；
3. 循环记忆强度上限；
4. 是否加入第二种实体威胁；
5. 终章追逐脱离常规规则的程度；
6. 是否提供“降低怪物灵敏度”的可访问性选项。
