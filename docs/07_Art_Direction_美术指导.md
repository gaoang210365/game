# 《回声病房》美术指导 / Art Direction

> **文档版本：** 0.1（初稿）
> **前置文档：** 01～06 号文档
> **文档日期：** 2026-07-14
> **文档状态：** 美术方向，供素材制作参考

---

## 1. 目的

本文档确定《回声病房》的整体美术风格、色彩语言、材质、光照、UI 视觉与素材制作规范，确保画面风格统一、可实现，并服务于心理恐怖体验。

---

## 2. 总体风格

采用：

```text
低多边形写实基础
+ PS2 / 早期 PC 恐怖游戏质感
+ 现代动态灯光与空间化声音
```

### 2.1 选择理由

- 与 Python / Panda3D 技术规模匹配；
- 降低模型与材质制作成本；
- 低精度视觉本身能制造不确定感；
- 更容易形成统一且可辨识的风格；
- 允许把资源集中在灯光、构图与声音上；
- 对中低配置电脑友好。

### 2.2 风格关键词

- 陈旧、潮湿、被遗弃；
- 冷硬的医疗环境；
- 低分辨率贴图带来的模糊不安；
- 强对比的光影；
- 克制而精准的恐怖细节。

## 3. 色彩语言

对应愿景文档第 11 节，统一色彩规范：

| 状态 | 主要色彩 | 用途 |
|---|---|---|
| 暂时安全 | 冷白、暗蓝、浅绿 | 护士站、门诊等相对安全区 |
| 潜在危险 | 锈黄、低饱和橙、灰黑 | 过渡区域、储物间 |
| 直接危险 | 暗红、深黑、过曝白 | 追逐、怪物出现 |
| 记忆片段 | 低饱和暖色、褪色棕 | 闪回、回忆 |
| 真相空间 | 冷白与红色同时出现 | 终章、404 |

### 3.1 色彩原则

- 每个区域有明确主色调，帮助玩家辨认状态；
- 危险色（暗红）不滥用，仅在真实威胁时强化；
- 谜题信息不能仅靠颜色区分（可访问性）；
- 记忆场景与现实场景通过色调明显区分。

## 4. 场景美术

### 4.1 一层门诊

- 相对完整但落满灰尘；
- 冷白日光灯（部分闪烁）；
- 挂号台、候诊椅、旧海报；
- 地标：大厅巨型挂钟。

### 4.2 二层住院部

- 潮湿、剥落的墙面；
- 绿色指示灯；
- 病床、输液架、隔帘；
- 404 号病房外观与其他病房有细微差异（暗示）。

### 4.3 地下手术区与太平间

- 最压抑区域；
- 手术室无影灯、金属器械；
- 太平间编号抽屉墙；
- 暗红应急灯。

### 4.4 材质规范

- 使用低分辨率但风格统一的贴图；
- 重复使用模块化墙、地、门、病床套件；
- 通过脏污、水渍、血迹贴花增加变化；
- 避免高精度写实材质，保持统一低多边形质感。

---

## 5. 角色美术

### 5.1 值夜护士

- 见怪物设计文档第 2 节；
- 面部遮挡是关键视觉；
- 输液架为标志性道具；
- 终章逐渐显现与主角相似的细节。

### 5.2 主角

- 第一人称为主，较少露出身体；
- 可在镜像、倒影中显现，用于叙事；
- 手部模型用于交互动画。

### 5.3 患者与院方

- 不以实体角色正面出现；
- 通过照片、绘画、录像间接呈现。

---

## 6. 光照设计

光照是本作最重要的美术手段之一。

### 6.1 原则

- 强对比明暗，大量阴影区域；
- 限制动态光源数量以保证性能；
- 静态环境尽量使用烘焙信息；
- 手电筒为主要动态光源；
- 应急灯、闪烁灯用于制造节奏与恐怖。

### 6.2 光照与情绪

- 安全区：均匀冷光；
- 过渡区：昏暗、局部光；
- 危险区：频闪、暗红、强逆光剪影；
- 终章：冷白与红色交替。

---

## 7. UI 视觉

### 7.1 原则

- 极简，减少常驻 HUD；
- 与医疗、旧设备风格呼应；
- 交互提示低调但清晰；
- 档案界面模拟病历、文件质感。

### 7.2 内容

- 交互提示图标；
- 物品/证据获取提示；
- 档案查看界面；
- 字幕（可调大小与背景）；
- 暂停与设置菜单。

---

## 8. 图像生成工具定位

`gpt-image-2` 可用于：

- 概念艺术与场景气氛参考；
- 海报、封面；
- 墙面海报、旧照片、患者绘画等叙事素材；
- UI 视觉方向探索；
- 低风险装饰纹理草案。

不直接依赖生成式图片制作：

- 必须精确拼接的主场景材质；
- 含关键谜题数字与文字的最终素材；
- 碰撞模型；
- 需要骨骼动画一致性的角色模型。

所有 AI 素材在发布前必须检查：许可、文字准确性、风格一致性、潜在瑕疵。

---

## 9. 素材制作规范

- 模型：低多边形，模块化，统一比例尺；
- 贴图：统一分辨率档位，命名规范；
- 目录结构遵循技术设计文档 assets 规范；
- 贴图与模型分离管理；
- 大型二进制资源需评估 Git LFS 或外部存储；
- 资源版本与命名保持一致，避免重复文件。
