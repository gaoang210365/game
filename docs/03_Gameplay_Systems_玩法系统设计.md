# 《回声病房》玩法系统设计 / Gameplay Systems

> **文档版本：** 0.1（初稿）
> **前置文档：** 01_Game_Vision、02_Story_Outline
> **文档日期：** 2026-07-14
> **文档状态：** 系统设计，供技术设计与实现参考

---

## 1. 目的

本文档定义《回声病房》的核心玩法系统、系统之间的关系，以及每个系统的设计目标、规则和边界，作为后续技术设计和编码的依据。

设计总原则：

- 所有系统必须服务于心理恐怖体验；
- 优先做少而扎实的系统，不追求功能数量；
- 系统之间通过明确的事件通信，避免相互硬编码；
- 玩家的失败应来自误判规则，而非系统不透明。

---

## 2. 系统总览

首版包含以下核心系统：

1. 玩家控制系统；
2. 手电筒系统；
3. 交互系统；
4. 物品与证据系统；
5. 门与容器系统；
6. 循环与空间变化系统；
7. 怪物感知与 AI 系统；
8. 恐怖事件系统；
9. 音频系统；
10. 存档与检查点系统；
11. UI 与字幕系统；
12. 设置与可访问性系统。

系统关系概述：

```text
玩家控制 → 产生噪声/视线/位置
        → 交互系统 → 物品、门、事件
        → 手电筒 → 影响可见性与怪物感知
循环系统 → 改变空间与线索状态
怪物AI ← 感知(声音/视线/光) ← 玩家与环境
恐怖事件 → 调度异象、追逐、脚本演出
音频 ← 所有系统的反馈来源
存档 ← 章节检查点与状态快照
UI/字幕 ← 交互、事件、音频提示
设置 → 影响控制、画面、音频、可访问性
```

## 3. 玩家控制系统

### 3.1 设计目标

提供有重量感但不迟钝的第一人称移动，让玩家在探索、躲避和追逐中都有清晰的操作反馈。

### 3.2 状态

玩家包含以下移动状态：

- 站立静止；
- 行走（默认，中等噪声）；
- 奔跑（高速，高噪声，受体力限制）；
- 蹲伏（低速，低噪声，可进入躲藏点）；
- 交互中（限制移动）；
- 被抓（触发失败流程）。

### 3.3 噪声模型

玩家动作产生噪声值，供怪物感知系统使用：

| 动作 | 相对噪声 |
|---|---|
| 蹲伏移动 | 很低 |
| 站立不动 | 极低 |
| 行走 | 中 |
| 奔跑 | 高 |
| 碰撞物品 | 瞬时高 |
| 开关门 | 瞬时中～高 |

噪声随距离衰减，可被环境声源部分掩盖。

### 3.4 体力

- 奔跑消耗体力；
- 体力不通过数值条直接显示，用呼吸声、镜头轻微起伏和速度下降反馈；
- 体力耗尽后强制回到行走；
- 停止奔跑后逐渐恢复；
- 体力用于限制玩家匀速跑过所有恐怖内容，而非模拟真实疲劳。

### 3.5 边界

- 不做跳跃平台玩法；
- 不做攀爬；
- 不做游泳；
- 镜头晃动强度必须可在设置中调整或关闭。

## 4. 手电筒系统

### 4.1 设计目标

手电筒既是照明工具，也是核心风险决策点：开灯看得清但更容易暴露，关灯安全但可能错过或遭遇另一类异常。

### 4.2 模式

- 普通照明；
- 不稳定/闪烁（剧情或低电量触发）；
- 特殊光照（用于显现特定痕迹，作为可选机制评估）。

### 4.3 电力规则

采用“过热/不稳定”而非永久耗尽：

- 持续使用会进入不稳定状态，光线闪烁；
- 短暂关闭后恢复稳定；
- 不设计会导致玩家永久陷入全黑的无解状态；
- 剧情关键段落可临时强制手电筒失效。

### 4.4 与其他系统关系

- 手电筒光锥可能提高怪物对玩家的感知；
- 部分异象仅在关灯时出现；
- 部分线索仅在开灯或特殊光照下可见；
- 光照状态作为事件触发条件之一。

---

## 5. 交互系统

### 5.1 设计目标

提供统一、可预期的交互反馈，让玩家清楚知道什么可以互动。

### 5.2 交互对象类型

- 可拾取物品；
- 可阅读线索；
- 门；
- 抽屉/柜子；
- 开关（电源、广播、设备）；
- 躲藏点；
- 剧情触发点。

### 5.3 交互规则

- 准星或高亮提示可交互对象；
- 交互距离有限；
- 交互时限制部分移动；
- 交互产生相应噪声；
- 交互结果通过音效、UI 和字幕反馈；
- 不使用无提示的像素级点击。

---

## 6. 物品与证据系统

### 6.1 分类

1. 关键道具：推进流程（钥匙、门禁卡、保险丝、404 钥匙）；
2. 证据线索：推进剧情理解（病历、录音、文件、绘画）；
3. 应急道具：提供有限反制（声源装置、一次性闪光、镇静剂）。

### 6.2 规则

- 背包保持轻量，无重量系统，无复杂合成；
- 关键道具必获，证据线索部分可选；
- 证据被查看后进入“档案”界面，可重复回看；
- 应急道具数量有限，鼓励谨慎使用；
- 物品获取与使用均有明确反馈。

### 6.3 与结局关系

- 已收集的关键证据数量决定玩家能否进入完整结局；
- 证据系统需记录收集状态并持久化到存档。

## 7. 门与容器系统

### 7.1 设计目标

门是恐怖体验的重要交互对象，需要多样的状态和反馈。

### 7.2 门状态

- 关闭 / 打开；
- 上锁（需要钥匙或权限）；
- 可从一侧临时堵住；
- 被怪物撞击；
- 缓慢推开 / 快速打开（影响噪声）。

### 7.3 容器

- 抽屉、柜子、储物箱；
- 可存放线索、道具；
- 部分容器兼作躲藏点；
- 打开容器产生噪声。

### 7.4 躲藏点

- 衣柜、床下、隔间；
- 进入后视野受限；
- 怪物可能靠近检查；
- 错误行为（噪声、过早离开）会暴露玩家；
- 躲藏不是绝对安全。

---

## 8. 循环与空间变化系统

### 8.1 设计目标

实现“熟悉空间中的不确定性”，让相同地点在不同循环呈现受控变化。

### 8.2 触发方式

- 主要触发：玩家穿过住院部指定防火门；
- 触发后推进“循环层”，改变日期、部分空间和线索状态。

### 8.3 变化类型

- 门牌与房间编号变化；
- 房间内部布局错位；
- 走廊长度/连接变化；
- 出现不同年代物品；
- 照片人物数量变化；
- 已开门重新上锁；
- 上一轮玩家动作被“重演”。

### 8.4 规则

- 变化基于状态机和预设规则，而非纯随机；
- 玩家应能通过观察学习规律；
- 关键地标保持可辨认，避免完全迷失；
- 循环变化需在存档中记录当前层与状态。

## 9. 怪物感知与 AI 系统

### 9.1 感知来源

- 玩家噪声（移动、开门、碰撞）；
- 手电筒光线；
- 视线内的玩家；
- 环境事件产生的声源。

### 9.2 AI 状态机

```text
巡逻 → 察觉异常 → 前往调查 → 发现玩家 → 追逐
→ 搜索最后已知位置 → 失去目标 → 返回/改变巡逻
```

### 9.3 公平性规则

- 不无提示瞬移到玩家身后；
- 不穿墙；
- 不永久知道玩家位置；
- 追逐前有声音或视觉预警；
- 追逐路线经过灰盒测试并提供决策点；
- 难度提升通过感知范围和搜索时间调整，不破坏规则一致性。

### 9.4 与循环结合

- 怪物在循环推进中逐渐“记住”玩家常用路线；
- 该记忆用于叙事，也用于渐进提升压力。

---

## 10. 恐怖事件系统

### 10.1 设计目标

统一调度不同强度的恐怖内容，控制节奏，避免玩家脱敏。

### 10.2 事件分级

- 一级：环境不安（灯闪、远处脚步、门把手转动）；
- 二级：视觉异象（镜像、人影、物品变化）；
- 三级：直接威胁（怪物经过、躲藏点被查、被迫关灯）；
- 四级：追逐（数量有限，各有教学或叙事目的）。

### 10.3 调度规则

- 事件通过触发器（区域、视线、时间、状态）激活；
- 系统记录近期强度，避免连续高压；
- 高压事件后安排释放期；
- 支持脚本化演出与条件化随机组合；
- 假警报与真实威胁需按比例混合。

---

## 11. 音频系统

### 11.1 设计目标

声音先于视觉提供信息，并能独立构成恐怖体验。

### 11.2 层级

- 环境底噪；
- 空间音源（方向、距离、遮挡近似）；
- 玩家反馈；
- 怪物标识音；
- 叙事音频；
- 少量音乐。

### 11.3 规则

- 支持 3D 音源与距离衰减；
- 怪物接近可通过“环境声消失”而非单纯增大音量表现；
- 相同音效避免机械重复；
- 提供主音量、环境、音效、语音、音乐独立调节；
- 重要语音配字幕。

---

## 12. 存档与检查点系统

### 12.1 设计目标

保证玩家进度安全，同时不破坏恐怖节奏。

### 12.2 规则

- 采用自动存档 + 章节检查点；
- 检查点设在关键谜题完成后；
- 失败从最近检查点恢复；
- 存档需记录：玩家位置、物品、证据、循环层、空间状态、事件进度；
- 恢复时不强制重复长篇阅读和已完成简单操作；
- 存档数据不包含任何敏感凭据。

---

## 13. UI 与字幕系统

### 13.1 设计目标

在不破坏沉浸感的前提下提供必要信息。

### 13.2 内容

- 交互提示；
- 物品/证据获取提示；
- 档案查看界面；
- 字幕与语音提示；
- 暂停菜单与设置。

### 13.3 规则

- 尽量减少常驻 HUD；
- 手电筒状态、体力等通过画面和音效隐性表达；
- 字幕支持开关、大小与背景；
- 关键音效可选提供视觉提示。

---

## 14. 设置与可访问性系统

对应愿景文档第 14 节，包含：

- 鼠标灵敏度、FOV；
- 镜头晃动、动态模糊开关；
- 奔跑按住/切换；
- 字幕设置；
- 亮度/黑位校准；
- 音频分类音量；
- 追逐难度选项；
- 避免仅用颜色传达谜题信息。

---

## 15. 系统优先级

首版实现优先级：

1. 玩家控制、交互、门、手电筒（基础可玩）；
2. 物品与证据、存档（流程完整）；
3. 怪物感知与 AI、恐怖事件（核心体验）；
4. 循环与空间变化（核心特色）；
5. 音频完善、UI/字幕、设置（体验打磨）。

---

## 16. 待确认事项

1. 手电筒是否加入特殊光照模式；
2. 体力是否完全隐藏数值；
3. 应急反制道具的种类与数量；
4. 循环次数是否固定；
5. 是否提供“低惊吓强度”可访问性模式；
6. 存档是单槽自动存档还是多章节槽位。
