# 《回声病房》制作计划 / Production Plan

> **文档版本：** 0.1（初稿）
> **前置文档：** 01～09 号文档
> **文档日期：** 2026-07-14
> **文档状态：** 制作计划，供开发排期参考

---

## 1. 目的

本文档定义《回声病房》从设计到发布的阶段划分、每阶段目标、交付物、验收标准与里程碑，确保项目在可控范围内推进，避免规模膨胀。

---

## 2. 制作总原则

- 先设计，后实现；
- 技术实验与内容制作分离；
- 灰盒验证节奏与动线，再投入正式美术；
- 固定首发范围，新增内容进入候选清单；
- 每阶段有明确验收标准；
- 关键系统尽早验证（AI、循环、打包）。

---

## 3. 阶段总览

```text
阶段0：设计文档          （当前，进行中）
阶段1：技术实验          （验证引擎与关键技术）
阶段2：灰盒关卡          （验证节奏与动线）
阶段3：核心系统          （玩法系统实现）
阶段4：内容制作          （美术、音频、剧情、事件）
阶段5：优化与发布        （打磨、打包、发布）
```

## 5. 阶段1：技术实验

### 目标
在不制作正式内容的前提下，验证关键技术可行性。

### 任务
- Panda3D 启动与窗口/全屏；
- 第一人称控制器（移动、视角）；
- 碰撞检测；
- 基础灯光与阴影；
- 手电筒动态光；
- 3D 音源与距离衰减；
- 模型与贴图加载；
- 简单 AI 状态机与路径移动；
- 存档读写原型；
- Windows 打包验证。

### 交付物
- 一组独立技术验证 demo（各自单独）；
- 技术实验结论记录。

### 验收标准
- 每项关键技术单独跑通；
- 打包可在目标 Windows 设备运行；
- 明确不可行项及替代方案。

---

## 6. 阶段2：灰盒关卡

### 目标
用简单几何体搭建三层医院，验证节奏与动线。

### 任务
- 一层门诊灰盒；
- 二层住院部灰盒（含循环点）；
- 地下手术区/太平间灰盒；
- 主要谜题动线放置；
- 追逐路线与决策点；
- 循环变化的可读性测试；
- 躲藏点分布。

### 交付物
- 可行走的灰盒关卡；
- 动线与节奏测试记录。

### 验收标准
- 玩家能在灰盒中完成主线动线；
- 追逐路线可行且有决策点；
- 循环变化可被玩家察觉但不致完全迷失。

---

## 7. 阶段3：核心系统

### 目标
实现玩法系统，形成可玩流程。

### 任务（按优先级）
1. 玩家控制、交互、门、手电筒；
2. 物品与证据、存档；
3. 怪物感知与 AI、恐怖事件；
4. 循环与空间变化；
5. 音频基础、UI/字幕、设置。

### 交付物
- 可玩的核心系统集成版本（灰盒）；
- 系统调试工具。

### 验收标准
- 玩家可在灰盒中体验完整核心循环；
- 怪物 AI 公平且可学习；
- 循环与空间变化可用；
- 自动存档不破坏进度。

---

## 8. 阶段4：内容制作

### 目标
加入正式美术、音频、剧情与恐怖事件。

### 任务
- 正式模型与贴图（低多边形套件）；
- 光照与后处理；
- 音效与叙事音频；
- 剧情道具与线索投放；
- 恐怖事件（A/B/C/D 全清单）；
- 三个结局；
- UI 视觉完善。

### 交付物
- 内容完整的可玩版本；
- 全流程可通关。

### 验收标准
- 全流程 60～90 分钟可通关；
- 三个结局可达成；
- 恐怖节奏符合分级设计；
- 关键真相可通过环境证据推断。

---

## 9. 阶段5：优化与发布

### 目标

### 任务

- Bug 修复；

- README 与开发说明；

### 交付物

## 11. 范围管理

### 首版固定范围
- 三层医院；
- 60～90 分钟流程；
- 1 主要怪物；
- 5～7 核心谜题；
- 2～3 追逐；
- 3 结局。

### 候选清单（首发后评估）
- 额外结局；
- 第二种实体威胁；
- 更多循环层；
- 更精细的角色表演；
- 关卡编辑工具；
- 其他平台。

超出首版范围的需求一律进入候选清单，不影响主线进度。

---

## 12. 风险与应对

| 风险 | 应对 |
|---|---|
| 规模膨胀 | 严格范围管理，候选清单机制 |
| AI 调试耗时 | 优先做调试工具与灰盒测试 |
| 美术产能不足 | 低多边形 + 模块化 + AI 概念参考 |
| 打包问题 | 阶段1 即验证打包 |
| 恐怖节奏失衡 | 事件系统记录强度，安排释放期 |
| 单人开发精力有限 | 分阶段交付，保证每阶段可停可续 |

---

## 13. 协作工具与流程

- 设计文档：docs/ 目录，Markdown；
- 代码：独立 GitHub 仓库（建议 echo-ward）；
- 素材：assets/，大型资源评估 Git LFS；
- 图像概念：gpt-image-2 生成参考，人工筛选；
- 版本控制：功能分支 + 主分支，避免直接提交敏感信息。
